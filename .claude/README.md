# .claude Directory Organization

This directory contains Claude AI's workspace for the ml4t.backtest project. All planning, reference, and work-in-progress documents belong here, NOT in the project root.

## Directory Structure

### planning/
Implementation plans, roadmaps, and analysis documents:
- `IMPLEMENTATION_PLAN.md` - Detailed development plan
- `FRAMEWORK_ANALYSIS.md` - Analysis of existing backtesting frameworks
- `ROADMAP.md` - Development milestones and timeline

### reference/
Key design documents and reviews:
- `DESIGN.md` - Original 1000-line design specification
- `ARCHITECTURE_REVIEW.md` - Critical review and feedback

### sprints/
Sprint-based development work (when active)

### Key Files
- `PROJECT_GUIDELINES.md` - Repository organization rules
- `README.md` - This file

## Important Notes

1. **NEVER place planning documents in the project root**
2. **User-facing documentation goes in `/docs/`**
3. **Reference DESIGN.md for any architectural questions**
4. **Follow PROJECT_GUIDELINES.md for all development**
